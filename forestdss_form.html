Start here – assign a DSS identifier

This ID links all instances across sections. Use the DSS acronym or a short slug (e.g. SILVA-DSS). It auto-fills into all new instances.

Decision Support System
A computational or conceptual system integrating data, models, and analytical or participatory methods to support decision-making in forest management. Look for: the system name, acronym, overall purpose, and any links to documentation or publications.
owl:Class → Decision support system
Software
The computational component implementing the DSS. Look for: software name, programming environment, operating system, accessibility (open/commercial), and any information about who developed it and when.
owl:Class → Software
Object property: uses software – links Decision support system → Software
Model
A representation of processes implemented within the DSS. Add one instance per distinct model or model type mentioned. Look for: model names, whether they are process-based or empirical, what forest processes they represent (growth, disturbance, carbon...), and at what spatial/temporal scale.
owl:Class → Model (and subclasses)
Object property: uses model – links Decision support system → Model
One per model type
Forest Management Strategy
Forest management strategies, practices, and silvicultural approaches the DSS can represent or evaluate. Look for: silvicultural systems described, tree species considered, harvesting methods, and spatial scale of management.
owl:Class → Forest management strategy
Object property: supports forest management objective – links DSS → Forest management strategy
Forest Ecosystem Service
Ecosystem services the DSS addresses, models, or produces indicators for. Add one instance per ES. Look for: which services are mentioned, how they are quantified (biophysical, monetary...), and what indicators or outputs represent them. Use the CICES hierarchy from broad (Section) to specific (Class).
owl:Class → Forest ecosystem service category
Object property: targets forest ecosystem service – links DSS / Case study → ES. Tip: fill Section first, then Division, Group, and Class cascade automatically. Add one instance per distinct ES represented in the DSS.
Planning Problem
The decision-making context the DSS is designed to address. Look for: time horizon (strategic/tactical/operational), spatial scale, whether decisions involve one or many decision-makers, and whether multiple objectives are considered.
owl:Class → Planning problem
Object property: targets planning problem – links Case study → Planning problem
Case Study
An empirical application of the DSS in a real or study context. Look for: where and when the DSS was applied, what forest type or biome, what scenarios were used, and what management frameworks (SFM, AFM, IFM) were applied.
owl:Class → Case study
Object property: has case study – links Decision support system → Case study
Input Data
Data sources and datasets used by the DSS. Add one instance per distinct data type or source. Look for: what data the DSS requires as inputs, where it comes from (field survey, remote sensing, administrative records...), and at what spatial/temporal resolution.
owl:Class → Data → Input data
Object property: uses data source – links DSS → Input data
Output Data
Results and indicators produced by the DSS. Look for: what outputs the DSS generates (maps, tables, charts, scenario comparisons), in what format, at what scale, and whether uncertainty is included.
owl:Class → Data → Output data
Object property: produces data output – links DSS / Model → Output data
Decision Support Technique
Analytical methods used within the DSS to process information and support decisions. Look for: whether the DSS structures problems, models alternatives, or optimizes solutions; and which specific MADM or MODM methods are used.
owl:Class → Decision support technique
Object property: uses technique – links DSS → Decision support technique
Uncertainty Evaluation
How the DSS handles uncertainty. Look for: whether uncertainty is explicitly assessed, which methods are used (Monte Carlo, sensitivity analysis...), and where uncertainty originates (input data, model structure, scenarios...).
owl:Class → Uncertainty evaluation
Object property: uses uncertainty evaluation – links DSS / Model → Uncertainty evaluation
Scenario
Future conditions or alternative states of the world used in the DSS analysis. Look for: climate scenarios (RCP, SSP, GWL), management scenarios, and the time horizon covered.
owl:Class → Scenario
Object property: uses scenario – links Case study → Scenario
Participatory Process
Structured processes involving stakeholders in DSS design or application. Look for: whether stakeholders were involved, how (workshops, surveys, co-design), who facilitated, and whether the process was one-time or iterative.
owl:Class → Participatory process
Lessons Learned
Insights derived from applying the DSS. Look for: explicit lessons, conclusions, or recommendations stated by the authors about what worked, what did not, and what they would recommend to others.
owl:Class → Lesson learned
Object property: documents lesson – links Case study → Lesson learned
Knowledge Management Processes
Knowledge management activities supported by or resulting from the DSS. Look for: whether the DSS supports knowledge creation, storage, transfer, or application, and what tools or platforms are used.
owl:Class → Knowledge management process
Object property: support knowledge management process – links Case study → KM process
Actor / Person
People involved in the DSS or case study as developers, researchers, decision-makers, stakeholders, or beneficiaries. Note: the DSS contact person for wiki purposes is recorded in the DSS section. This section covers actors relevant to understanding who developed the system and who participated in or was affected by its application.
owl:Class → Person (Individual)
Object properties: has actor, has contact person – link DSS / Case study → Person
